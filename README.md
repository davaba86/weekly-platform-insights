# Weekly Platform Insights

This repository contains my weekly technical notes and experiments related to platform engineering, cloud architecture, and infrastructure automation.

Each week I document something I built, tested, or learned while working with technologies such as AWS, Terraform, Kubernetes, and CI/CD pipelines.

The goal is simple:

- document practical solutions
- keep a public record of experiments and lessons
- share small, focused ideas instead of long tutorials

Some weeks focus on architecture decisions, others on implementation details or lessons learned while building things.

## Structure

Each week has its own folder.

Example:

2026-11/
- README.md
- diagrams
- code snippets
- configuration examples

Inside each weekly folder you will find a short write-up describing the problem, the approach, and the implementation.

## Topics you may find here

Examples include:

- Terraform in multi-account AWS environments  
- GitHub Actions pipelines for infrastructure  
- OIDC authentication between GitHub and AWS  
- Kubernetes and GitOps experiments  
- infrastructure design decisions  
- lessons learned from building platforms

## Why this repository exists

I wanted a single place where I can document ideas and experiments in a simple format.

Instead of keeping notes privately, this repository acts as a public log of the things I am building and exploring.

Many of the entries also serve as the basis for:

- LinkedIn posts
- YouTube explanations
- deeper technical write-ups
